\begin{frame}[t]{init (リモート)}{使ってみる}

  remote repository を用意する
  \vspace{4ex}

  \onslide*<1>{
    remote repository の作成
    \vspace{2ex}

    \$ mkdir \~{}/remote\_rep

    \$ cd \~{}/remote\_rep

    \$ git init {\dhyphen}bare
  }

  \onslide*<2>{
    remote repository のエイリアス作成
    \vspace{2ex}

    \$ cd \~{}/local\_rep

    \$ git remote add origin \~{}/remote\_rep
  }
\end{frame}


\begin{frame}[t]{git init コマンド補足 (リモート)}{使ってみる}

  \begin{itemize}
  \item 多くの git server では URI を ssh や https で指定するが

    それはネットワーク的な問題

    repository 自体はこの用法で作成している事も多い
    \vspace{2ex}

  \item この方法で作成した remote repository は

    workspace が無いので

    普通の local repository としては使用不可

    その他、git の各種デフォルト設定が異なっている
  \end{itemize}

\end{frame}
